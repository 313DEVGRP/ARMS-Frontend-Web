A-RMS New Feature : 상태 관리

상태 관리는 고객사의 고유한 요구사항 상태관리와 ALM의 이슈 상태를 맵핑(연결)하는 기능을 제공합니다.

A-RMS 상태별 ALM 상태 매핑

A-RMS 상태를 동적으로 관리합니다.

A-RMS 상태와 ALM 이슈 상태를 매핑하여 요구사항을 관리할 수 있습니다.

A-RMS의 통계 수집을 위해서 4가지 상태 카테고리와 A-RMS 상태 간 매핑이 필요합니다.

ALM 서버 선택 → (클라우드 지라)프로젝트 선택 → (클라우드 지라)이슈 유형 선택 → 상태 매핑

ALM 서버 선택

1. ALM 서버 선택
  • 선택된 서버 : 선택되지 않음
    선택된 프로젝트 : 선택되지 않음
    선택된 이슈유형 : 선택되지 않음
    • 클라우드 지라의 경우 "프로젝트" 항목 하위의 이슈 유형을 선택하시면 해당 이슈 유형별 상태를 매핑할 수 있습니다.
